Original Issue

After building with PyInstaller, waypoint.exe would launch but display:

GET http://127.0.0.1:5000/ net::ERR_HTTP_RESPONSE_CODE_FAILURE 404 (Not Found)

üîç Root Cause Analysis

Initial Symptoms:
  • Script version (python app.py) worked perfectly
  • Bundled exe (waypoint.exe) returned 404
  • Server started and listened on port 5000
  • But no files could be served
Investigation Process:
  • Checked if modern-ui.html existed in bundled resources
  • Confirmed BASE_DIR pointed to PyInstaller temp directory
  • Discovered waypoint.spec was missing data files
Root Cause Identified:

waypoint.spec only included config.yaml in the datas section. Modern-ui.html and assets folder were NOT being bundled into the executable.

‚úÖ Solution Implemented

Fixed waypoint.spec

Before:

datas=[('config.yaml', '.')]

After:

datas=[
    ('config.yaml', '.'),
    ('modern-ui.html', '.'),
    ('assets', 'assets'),
]

This ensures PyInstaller bundles all necessary files into the executable.

üéì Lessons Learned

Key Insights

  • PyInstaller data files: Must be explicitly listed in waypoint.spec
  • BASE_DIR vs DATA_DIR: BASE_DIR points to temp directory (read-only), DATA_DIR points to exe directory (writable)
  • Testing strategy: Always test the built exe, not just the script
  • Port conflicts: Multiple server instances can cause mysterious timeouts
